\subsection{Customer Clustering Using K-Means}

[Somewhere I need to specify that by customer, I actually mean customer + facility combination (one customer can have several facilities, and these are treated uniquely)]

Since there are many customers in the electricity consumption data, it is relevant to group customers with similar consumption patterns. The idea is to identify groups where the similarity within the group is higher than that among the groups \cite{khan_ensemble_2021}. K-Means clustering is one of the most popular algorithms for solving this problem in a simple way. K-Means is an unsupervised learning algorithm, where the main idea is to select $k$ centroids, one for each cluster, and place them as far from each other as possible. The next step is to associate each point in the dataset with the closest centroid. After associating all points, the $k$ centroids are recalculated by averaging the points within each cluster. This process is repeated until the centroids no longer move. Formally, the algorithm aims at minimizing the squared error function $W(S,C)$ defined in \autoref{equ:k-means} \cite{kodinariya_review_2013}.

\begin{equation}\label{equ:k-means}
    W(S,C) = \sum_{k=1}^{K} \sum_{i \in S_k}^{} \left\| y_i - c_k \right\|^2
\end{equation}

In the equation, $S$ is a partition of $k$ clusters represented by observations in the form of vectors $y_i$. Each cluster $S_k$ is non-empty and non-overlapping, and has a centroid $c_k$ ($k = 1,2,\dots,K)$.

In these experiments, K-Means clustering is applied to define customer groups based on their electricity consumption. The elbow method is used to find an optimal number of $k$, i.e. the number of clusters. The elbow method is a visual method, where different values of $k$ are plotted against the variance within the clusters \cite{cui_introduction_2020}. As the value of $k$ increases, the average variance becomes smaller. The goal is to find a value of $k$ for which the variance stops decreasing sharply, that is, the elbow of the graph. \autoref{fig:elbow_plot} shows the elbow plot for determining the optimal number of customer groups, in this case $k=3$. This results in three customer groups; low consumption, medium consumption, and high consumption. Each customer in the dataset is assigned a label based on the cluster it belongs to, and this label becomes an input feature for the forecasting models. \section{Ensemble Learning}

Forecasting is performed using different decision tree-based ensemble learning models. Decision trees classify data items by asking a sequence of questions about the items' features. The nodes of the tree contain questions, and for each possible answer, every internal node points to one child node. The sorting of an item into a class begins from the root of the tree, and according to the item's answer to the questions, it follows the path to a leaf node \cite{kingsford_what_2008}. [Possibly a diagram of a decision tree] One large advantage with decision trees is that they are considerably easier to interpret than neural networks, as they follow logical rules instead of having connections with numeric weights between nodes. Decision trees perform well for a few highly relevant attributes, but not as well if there are many attributes with complex interactions. Ensembles of decision trees can be employed to increase accuracy when the training data is more complex and changing \cite{kotsiantis_decision_2013}. Decision tree-ensembles are considered among the most reliable and precise methods for regression and classification tasks, and they have been shown to be effective for predicting energy consumption \cite{moon_advancing_2024}. However, one disadvantage is that the interpretability decreases when multiple classifiers are involved, making it more difficult to understand the reasoning behind the ensemble's decision \cite{kotsiantis_decision_2013}. This creates a need for model explainability outside the ensemble's inner workings, which in this thesis is addressed by applying explainable AI. [move to theory?]

Five decision tree-based ensemble models are selected as forecasting models, as in \cite{moon_advancing_2024}; random forest (RF), gradient boosting machine (GBM), extreme gradient boosting machine (XGBoost), light gradient boosting machine (LightGBM), and categorical boosting (CatBoost). [GBM and CatBoost were best, so focus on them first]

Gradient boosting machines are a family of ML algorithms where the learning process consecutively fits a new tree with the goal of reducing the errors of the previous one. This is done by constructing new base-learners that are maximally correlated to a decreasing loss function. The loss function can be of various kinds, which makes GBM highly flexible for different data-driven tasks. It is up to the model designer to choose the most appropriate loss function \cite{natekin_gradient_2013}. GBM is able to learn complex patterns and interactions in the data due to its iterative learning process \cite{moon_advancing_2024}. [Possibly a diagram showing its structure]

CatBoost is a gradient boosting algorithm that uses a technique called \emph{ordered boosting} to reduce prediction shift caused by target leakage which occurs in other gradient boosting algorithms. CatBoost also encodes categorical features uniquely, by applying a technique called \emph{ordered target statistics} to handle high cardinality features \cite{prokhorenkova_catboost_2018}. Because of this, CatBoost is able to achieve high accuracy, especially when the data contains a significant number of categorical features \cite{moon_advancing_2024}. \section{Evaluation Metrics}
To evaluate the predictive performance of the forecasting models, three statistical metrics are used; mean absolute percentage error (MAPE), coefficient of variation of the root mean squared error (CVRMSE), and normalized mean absolute error (NMAE). MAPE is one of the most common metrics to evaluate forecasting accuracy, due to its scale-independence and interpretability. It defines the average of the absolute percentage errors, as in \autoref{equ:mape}. One disadvantage to be aware of is that MAPE yields infinite or undefined values when the actual values are zero or close to zero \cite{kim_new_2016}. [Maybe use RSME and MAE instead?]

These metrics express the performance in error percentage, making them intuitive to interpret.

\begin{equation}\label{equ:mape}
    MAPE = \frac{1}{n} \times \sum \left| \frac{Y_t - \hat{Y}_t}{Y_t} \right| \times 100
\end{equation}

\begin{equation}\label{equ:cvrmse}
    CVRMSE = (\sqrt{\frac{1}{n} \times \sum (Y_t - \hat{Y}_t)^2}) / \ddot{Y} \times 100
\end{equation}

\begin{equation}\label{equ:nmae}
    NMAE = (\frac{1}{n} \times \sum \left| Y_t - \hat{Y}_t \right|) / \ddot{Y} \times 100
\end{equation}

In the equations, $Y_t$ denotes the actual observed value at time $t$, while $\hat{Y}_t$ denotes the predicted value at time $t$. $\ddot{Y}$ represents the mean of all observed values, and $n$ is the total number of prediction instances.

To evaluate the models in a more comprehensive way, the harmonic mean (HM) is calculated based on these three metrics. The harmonic mean will favor models that perform well in all three metrics, while models that perform exceptionally well in one metric but poorly in the others will be valued less. This is due to the harmonic mean being effective at mitigating extreme outliers, making it suitable for combining differing metrics \cite{moon_advancing_2024}. The HM is calculated as in \autoref{equ:harmonic-mean}.

\begin{equation}\label{equ:harmonic-mean}
    HM = \frac{3}{(1/MAPE) + (1/CVRMSE) + (1/NMAE)}
\end{equation}
